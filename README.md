# 🧠 AI Business OS

دستیار اجرایی هوشمند کسب‌وکار از طریق گفتگو در تلگرام.

سیستم چندکارفرمایی (Multi-Tenant) که با هوش مصنوعی، فروش/هزینه/مشتری/کارمند/انبار را از طریق مکالمه‌ی طبیعی مدیریت می‌کند.

---

## 🏗 معماری

```
کاربر (تلگرام)
      ↓
هندلر تلگرام  ──→  کنترل دسترسی (Tenant + اشتراک)
      ↓
Orchestrator (حلقه‌ی agentic)
      ↓
موتور AI (OpenRouter → Gemini)  ──→  تشخیص قصد + انتخاب tool
      ↓
Dispatcher  ──→  سرویس ماژول‌ها (مشتری/هزینه/...)
      ↓
دیتابیس PostgreSQL (جداسازی با tenant_id)
```

**نکته‌ی کلیدی:** مدل AI خودش داده ثبت نمی‌کند. فقط *قصد* کاربر را می‌فهمد و tool درست را صدا می‌زند. اجرای واقعی با کد قابل‌اعتماد ماست. به همین دلیل یک جمله را به ده شکل می‌توان گفت و سیستم می‌فهمد.

## 🚀 راه‌اندازی روی Railway

### ۱. آماده‌سازی کلیدها
- **توکن تلگرام:** از [@BotFather](https://t.me/botfather) یک ربات بساز
- **کلید OpenRouter:** از [openrouter.ai/keys](https://openrouter.ai/keys) بساز
- **آیدی عددی خودت:** از [@userinfobot](https://t.me/userinfobot)

### ۲. دیپلوی
1. این پروژه را روی یک ریپازیتوری GitHub بگذار
2. در Railway: `New Project` → `Deploy from GitHub repo`
3. یک سرویس **PostgreSQL** اضافه کن (`New` → `Database` → `PostgreSQL`)
   - Railway خودکار متغیر `DATABASE_URL` را به سرویس وصل می‌کند
4. در تب `Variables` این‌ها را اضافه کن:
   ```
   TELEGRAM_BOT_TOKEN=...
   OPENROUTER_API_KEY=...
   OPENROUTER_MODEL=google/gemini-2.5-pro
   SUPER_ADMIN_IDS=آیدی_عددی_تو
   ```
5. دیپلوی خودکار شروع می‌شود ✅

### ۳. تست
در تلگرام به ربات `/start` بزن.

## 🔧 افزودن ماژول جدید

الگوی توسعه ساده است:
1. مدل را در `database/models/business.py` اضافه کن
2. سرویس را در `modules/<نام>/service.py` بنویس
3. tool را در `ai/tools.py` تعریف کن
4. یک خط به `ai/dispatcher.py` اضافه کن

همین. بقیه‌ی سیستم خودکار کار می‌کند.

### فاکتور و فروش

## ⚠️ نکته‌ی مهم هنگام به‌روزرسانی دیتابیس

به‌خاطر افزوده‌شدن ستون‌ها و جداول جدید (SearchTask، فیلدهای جدید Tenant و...)،
`create_all` ستون‌های جدید را به جداول قدیمی **اضافه نمی‌کند**.

برای اعمال تغییرات، یک بار در کنسول PostgreSQL ریلوی این را اجرا کنید:

```sql
DROP SCHEMA public CASCADE;
CREATE SCHEMA public;
```

سپس سرویس را Restart کنید تا جداول از نو با ساختار کامل ساخته شوند.
⚠️ این کار همه‌ی داده‌ها را پاک می‌کند — قبلش از داده‌ها اکسل بگیرید.

## 🆕 آپدیت ۷ — سیستم نقش و لینک دعوت

### نقش‌ها
پنج نقش: کارفرما، کارمند، همکار، مشتری، پارتنر.
- هر نقش دسترسی متفاوت به ابزارها دارد
- ربات با هر نقش لحن متفاوت دارد
- فعلاً کارفرما و کارمند کامل فعال‌اند؛ بقیه پایه‌ای

### اشخاص
- کارفرما اشخاص را ثبت می‌کند: «رضا رو به‌عنوان کارمند ثبت کن»
- هر شخص ابتدا «ثبت‌شده ولی متصل‌نشده» است

### لینک دعوت
- کارفرما لینک دعوت می‌سازد: «لینک دعوت برای رضا بساز»
- شخص روی لینک کلیک می‌کند → آیدی تلگرامش خودکار وصل می‌شود
- ویژگی‌های امنیتی: قفل به شخص، انقضا، رمز، سقف استفاده
- پیش‌فرض هوشمند: کارمند → قفل+۷روز، مشتری → چندبارمصرف

### نکته‌ی فنی
- ملاک شناسایی: آیدی عددی تلگرام (نه یوزرنیم)
- یوزرنیم ربات هنگام راه‌اندازی خودکار گرفته می‌شود

### ویس

(مدل voxtral-mini-transcribe، فرمت wav).

- پیام مستقیم: «به علی بگو فاکتورش آماده‌ست»
